# Kali Linux Network and Security Auditing Scripts

This repository contains basic shell scripts for system auditing, connection tracking, and firewall management on Kali Linux.

## Why files must end with a new line?
According to the POSIX standard, a line is defined as a sequence of zero or more non-newline characters followed by a terminating newline character. If a file doesn't end with a newline, some older tools, compilers, or interpreters might ignore the last line completely, and commands like `wc -l` will not count it correctly.

## Requirements
* Allowed editors: `vi`, `vim`, `emacs`
* All scripts are exactly 2 lines long
* Executable permissions are granted to all scripts
